# Десктопная оболочка

Два запланированных варианта, оба поверх существующих биндингов:

1. **Tauri-приложение** (ближний срок): UI на webview, движок — либо
   `abbrev-wasm` внутри webview, либо `abbrev-core` напрямую в Rust-процессе
   Tauri (предпочтительно: нет границы WASM, общий бинарь). Сценарий —
   «блокнот с подсказками»: поле ввода со строкой кандидатов, удобно для
   демо, сбора фидбека и тюнинга весов на больших лексиконах.
2. **Системный ассистент в духе lay-public** (дальний срок): перехват ввода на
   уровне ОС (evdev/AT-SPI на Linux), replay исправленного текста. Отдельный
   бинарь на `abbrev-core`, без webview. Требует прав на чтение ввода —
   строго opt-in.

Контракт движка тот же: `suggest / accept / forms_of_lemma / *_history`.
