# Исследовательская база (конспект)

Конспект обзора литературы, на котором основаны архитектурные решения.
Полный обзор с источниками — в исходном исследовании проекта.

## Главные выводы

1. **Готовой литературы по русской «аббревиатурной IME» нет.** Решение —
   гибрид двух зрелых направлений: русская морфология (лексиконы, парадигмы,
   суффиксы) + error-tolerant ввод (spelling correction, pinyin-IME).
2. **Символьная генерация кандидатов + лёгкое ранжирование, не end-to-end
   нейросеть.** Даже современные нейро-IME (PinyinGPT и далее) показывают, что
   сокращённый ввод остаётся существенно труднее полного; чистая генерация —
   плохой первый решатель. Нейросети — как реранкер второй очереди.
3. **Архитектурный паттерн из spelling correction / pinyin-IME:** similarity
   keys / индексы → noisy-channel / взвешенный ред. дистанс → реранжирование
   частотой, контекстом, пользовательскими приорами; top-K-оценка.
4. **Русская специфика:** ранжировать надо поверхностные словоформы, а не
   леммы (работы по сегментации словоформ); обратная индексация по окончаниям
   критична (линия MyStem/Сеголовича — угадывание неизвестных слов по
   словарным окончаниям); главная боль ML-сегментации — незнакомые корни.
5. **Аналог гласных-пропусков** — иврит/арабский (восстановление огласовок):
   лексикон + дешёвые правки гласных + контекст восстанавливают полную форму.

## Перенесённые в код решения

* дешёвые вставки гласных и `ь/ъ` в ред. дистансе (Brill–Moore; Ahmad–Kondrak —
  веса можно доучивать из логов принятий);
* скелетные ключи (Pollock–Zamora) → согласный скелет-индекс;
* реранжирование частотой и контекстом (Sorokin/SpellRuEval, Chen–Lee, CHIME);
* top-1/top-3/латентность/keystroke-savings как обязательные метрики;
* приватный локальный лог принятий (ibus-typing-booster, lay-public).
