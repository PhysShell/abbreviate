# ADR-0001: Rust-ядро + тонкие платформенные оболочки

Статус: принято · Дата: 2026-06-12

## Контекст

Нужны Android IME (первая цель), затем iOS, десктоп и web. Логика
(лексикон, индексы, ред. дистанс, ранжирование, персонализация) одна на всех;
требования — офлайн, миллисекундная латентность, приватность.

## Решение

Вся доменная логика — в одном Rust-крейте `abbrev-core`. Платформенные оболочки
тонкие: ввод, отрисовка строки подсказок, хранение блобов (лексикон, история),
настройки. Контракт между ними — узкий API движка
(`suggest / accept / forms_of_lemma / export_history / import_history`).

## Альтернативы

* **Логика на каждой платформе (Kotlin + Swift + TS)** — N реализаций одного
  движка, расхождение качества и бенчмарков. Отклонено.
* **Kotlin Multiplatform / Flutter с движком внутри** — привязывает ядро к
  экосистеме UI; IME-оболочки всё равно нативные (InputMethodService,
  Keyboard Extension). Отклонено.
* **Сервер** — противоречит офлайн/приватность-требованиям IME. Отклонено.

## Последствия

* Один набор тестов и бенчмарков на все платформы; качество движка не зависит
  от фронтенда.
* Цена — граница FFI (ADR-0003) и дисциплина sans-IO (ADR-0002).
