# ADR-0004: Лексикон — версионированный артефакт; TSV сейчас, бинарный формат потом

Статус: принято · Дата: 2026-06-12

## Контекст

Лексикон (10⁵–10⁶ словоформ с леммами и частотами) собирается из OpenCorpora,
Wiktionary и частот НКРЯ. На устройстве он только читается; строить его на
устройстве нельзя (время старта, память).

## Решение

1. Лексикон — **артефакт конвейера** `tools/lexicon-builder`, собирается на
   машине разработчика/в CI, версионируется, кладётся в assets/bundle оболочки.
2. **Формат v0 — TSV** `form\tlemma\tfreq`: диффабельно, отлаживаемо, парсится
   ядром без зависимостей. Индексы строятся при старте движка.
3. Когда профилирование на устройстве покажет пределы (старт/память),
   формат меняется на **бинарный v1** (предсобранные индексы, mmap, FST/DAWG).
   Точка замены одна — загрузчик `Lexicon` и `PrefixMap`; API движка не меняется.

## Последствия

* Сейчас: простота и прозрачность; цена — построение индексов при старте
  (приемлемо до ~10⁵–10⁶ форм, проверяется бенчмарком).
* Граница совместимости — строка версии в заголовке артефакта (добавить при
  переходе на v1); ядро обязано отвергать незнакомые версии.
