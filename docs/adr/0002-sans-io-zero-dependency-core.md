# ADR-0002: Sans-IO ядро без зависимостей

Статус: принято · Дата: 2026-06-12

## Контекст

Ядро должно компилироваться под aarch64-android, iOS, wasm32 и десктоп,
укладываться в лимиты памяти iOS Keyboard Extension и давать детерминированные,
воспроизводимые результаты для бенчмарков.

## Решение

1. **Sans-IO:** `abbrev-core` не открывает файлы, не ходит в сеть, не читает
   часы и не создаёт потоки. Лексикон приходит строкой/байтами, история уходит
   блобом. Все побочные эффекты — на стороне оболочки/биндингов.
2. **Ноль runtime-зависимостей:** только `std`. Сериализация — простые
   TSV-форматы, парсеры свои.

## Последствия

* Тесты ядра мгновенные и герметичные; та же сборка идёт в UniFFI, WASM и CLI
  без feature-гимнастики.
* Цена — самописные мелочи (TSV-парсер) и отказ от удобных крейтов в ядре.
  В биндингах и инструментах зависимости разрешены (uniffi, wasm-bindgen,
  serde_json — уже используются).
* Потокобезопасность обеспечивает обёртка (`Mutex` в `abbrev-ffi`), не ядро.
