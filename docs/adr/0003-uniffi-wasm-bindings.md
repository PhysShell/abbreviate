# ADR-0003: UniFFI + wasm-bindgen как граница FFI

Статус: принято · Дата: 2026-06-12

## Контекст

Одно Rust-ядро должны потреблять Kotlin (Android IME), Swift (iOS),
JS/TS (web, Tauri-webview) и Python (исследования/скрипты).

## Решение

* **UniFFI** (`crates/abbrev-ffi`, proc-macro-режим): генерирует Kotlin/Swift/
  Python-биндинги из одного объявления `AbbrevEngine`. cdylib → Android `.so`,
  staticlib → iOS `.a`. Обёртка держит `Mutex` — колбэки IME многопоточны.
* **wasm-bindgen** (`crates/abbrev-wasm`): web и webview-десктоп. Подсказки
  через границу — JSON-строкой: стабильно, без тонкостей маршалинга структур.

## Альтернативы

* **Ручной C-ABI + JNI/Swift-обёртки** — максимум контроля, но ручная
  поддержка трёх биндингов и класс ошибок, который UniFFI устраняет. Отклонено.
* **flutter_rust_bridge / KMP-обёртки** — привязка к конкретному UI-стеку,
  тогда как оболочки IME всё равно нативные. Отклонено.
* **gRPC/HTTP к локальному процессу** — не работает в песочницах мобильных
  IME. Отклонено.

## Последствия

* Новая платформа = новая тонкая оболочка поверх уже существующих биндингов.
* Двойная поверхность (Record/Object у UniFFI, JSON у WASM) — цена за
  идиоматичность с обеих сторон; контракт зато один: API `abbrev-core`.
